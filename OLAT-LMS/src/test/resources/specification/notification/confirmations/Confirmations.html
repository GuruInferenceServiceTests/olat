Bestätigungen

Eine Bestätigung in OLAT ist eine vom System definierter Text, welcher über eine Aktion, von der man betroffen ist oder die man ausgelöst hat, per E-Mail informiert.

In folgenden Fällen erhält man als Student eine Bestätigung:

  • Wenn der Student seine Aufgabe in den Abgabeordner des Aufgabebausteins lädt
  • Wenn der Dozent die Korrektur in den Rückgabeordner des Aufgabebausteins lädt
  • Wenn der Dozent eine Musterlösung in den Musterlösungsordner lädt
  • Wenn der Dozent eine Bewertung im Bewertungsteilbaustein einträgt
  • Wenn der Dozent eine Bewertung in den Bewertungsbaustein einträgt
  • Wenn der Dozent den Studenten in eine Lerngruppe ein- oder austrägt

In folgenden Fällen erhält der Dozent eine Bestätigung:

  • Falls er seine Logfiles archivieren will, sobald die Archivierung der Logfiles abgeschlossen ist

In folgenden Fällen kann der Dozent wählen, ob er eine Bestätigung erhalten will:

  • Wenn der Student eine Aufgabe in den Abgabeordners des Aufgabebausteins lädt
  • Wenn der Student einen Test absolviert hat
  • Wenn ein Student ein Thema in der Themenvergabe aus- oder abwählt

Zu entfernende Bestätigungen

Folgende Bestätigung sind aus dem Code zu entfernen.

OrtIST-ZustandSOLL-ZustandBegründung
Einschreiben Der Benutzer, der sich eingeschrieben hat, erhält eine Bestätigung. Keine Bestätigung Wir gehen davon aus, dass der Benutzer weiss, was er getan hat.
Der Benutzer, der sich ausgetragen hat, erhält eine Bestätigung Keine Bestätigung Wir gehen davon aus, dass der Benutzer weiss, was er getan hat. Zudem erhält man momentan beim Verlassen einer Arbeitsgruppe keine Bestätigung. Dies soll weiterhin so bleiben und wäre somit konsistent.
Test Die Testteilnehmer werden bei Änderungen im Test informiert. Keine Bestätigung Mail in der heutigen Art ist nicht sehr verständlich, zudem kann das Changelog gelöscht werden. Changelog kann weiterhin im GUI angeschaut werden.
Arbeitsgruppe Besitzer und Mitbesitzer erhalten eine Bestätigung, wenn eine Arbeitsgruppe gelöscht wird. Keine Bestätigung Da die Teilnehmer nicht informiert werden und die Mitbesitzer in der Regel untereinander Aktion absprechen sollten, liegt die Verantwortung der Kommunikation beim Löscher. Über das Löschen der Lerngruppen und Rechtegruppen wird momentan auch nicht benachrichtigt.
Themenvergabe Wenn ein Thema gelöscht wird, erhält der Themenverantwortliche eine Bestätigung. Keine Bestätigung Man kann das Thema nicht wiederherstellen, sondern muss das Thema neu erstellen. Zudem wäre die Bestätigung über das Löschen des Themas v.a. für die Studenten interessant und die erhalten keine Bestätigung.
Der Benutzer erhält über die Wahl eines Themas eine Bestätigung. Keine Bestätigung Wir gehen davon aus, dass der Benutzer weiss, was er getan hat.
Der Benutzer erhält über die Abwahl eines Themas eine Bestätigung. Keine Bestätigung Wir gehen davon aus, dass der Benutzer weiss, was er getan hat.
Kurswizard Der Kursersteller erhält ein Bestätigungs/Informations-E-Mail nach der Erstellung eines Kurse mit dem Kurswizard. Keine Bestätigung Kurs im GUI ersichtlich. Der Kurswizard und der Kurseditor sollte so einfach sein, dass es keine lange Erklärungsmail braucht. Ev. wird der Kurswizard mit dem CampusCourse Projekt und Refactoring obsolet.
Kurs beenden Sobald ein Kurs beendet wird, werden die Kursbesitzer informiert. Keine Bestätigung Studenten werden momentan nicht darüber informiert. Kommunkation über Beendung liegt beim Beender. Mit der SAP-Anbindung sollte klar werden, welche Kurse aktuell sind.
Home/ Zentraldruckauftrags-Portlet Eine Auftrag kann an die Zentraldruckstelle versendet werden, damit ausgewählte Skripts ausgedruckt werden. Portlet entfernen Braucht es diese Funktion überhaupt noch? Druckereileiter informieren, falls sie gestrichen wird.
Kurs/ Aufgabebaustein Beim Hochladen einer Datei in den Abgabeordner wird der Benutzer über das Hochladen der eigenen Datei informiert, sofern dies im Kurseditor aktiviert wurde. Keine Bestätigung Wir gehen davon aus, dass der Benutzer weiss, was er getan hat. Jedoch soll das GUI so angepasst werden, dass der Benutzer die Dateien sieht, die er hochgeladen hat.
