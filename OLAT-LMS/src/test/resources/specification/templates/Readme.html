README

Dieses Readme-File soll beschreiben, wie die verschiedenen Elemente bzw. Stile und die Templates für die Spezifikation im Concordion verwendet werden können.

Hinweise

  • Die Dateien müssen UTF-8 codierts sein (Rechtsklick auf Datei, Properties klicken und unter File decoding sicherstellen, dass UTF-8 ausgewählt ist).
  • Die Dateinamen (Html-Files und Bilder) sind immer auf Englisch, möglichst kurz und aussagekräftig und ohne Unterstrich oder Bindestrich.
  • Bilder immer mittels CamelCase beschriften.
  • Die Akzeptanztests mittels CamelCase beschriften.
  • Damit die Krümmelpfade funktionieren, müssen die Ordner klein geschrieben sein, das erste Dokument muss gleich heissen wie der Ordner und der erste Buchstabe des Dokuments muss gross geschrieben sein.
  • Folgende Struktur einhalten
  • Im Overview.html findet man eine Übersicht. Im Context.html sind der Systemkontext und die Metriken enthalten. Die User Stories und zugehörigen Akzeptanztests sind eine Ebene tiefer. Es können mehrere User Stories Dateien vorhanden sein innerhalb eines userstory Ordners, aber auch mehrere userstory Ordner mit einzelnen User Stories Dateien. Ebenfalls kann ein userstory Ordner mehrere UserstoryScenarios enthalten.
  • Solange diese Ordnerhierarchie eingehalten wird, muss der Pfad zum Stylesheet (CSS-File) nicht angepasst werden.
  • Das Inhaltsverzeichnis entspricht der Ordner- und Dateienhierarchie. Der Title-Tag innerhalb des head-Tags wird als Eintrag für das Inhaltsverzeichnis genommen.
  • Beim Mouseover auf die Inhaltsverzeichniseintrage erscheint ein Hover mit den Einträgen der H-Tags innerhalb des jeweiligen Dokuments.
  • Einträge, die geändert bzw. gemacht werden müssen, sind in den Templates mit Grossbuchstaben und eckigen Klammern gekennzeichnet. Um diese "Platzhalter" noch mehr hervorzuheben, sind sie auf englisch notiert.
  • Im Commit-Text, jeweils ORID-#: angeben und die Changes möglichst treffend beschreiben.

Folgende Templates entsprechen folgenden Dateien im Notification-Beispiel:

TemplatesNotification

  • Will man eine neue Seite erstellen, kopiert man die gewünschte Template-Seite samt Ordner an die richtige Stelle und gibt ihr (und dem Ordner) einen neuen Namen. Um ein ganz neues Thema zu spezifizieren, kopiert man den Ordner "overview" samt Inhalt auf die gleiche Ebene wie der "notification" Ordner, also direkt unterhalb des "specification" Ordners, und passt die Namen aller Dateien und Subordner an. Will man nur eine neue Userstory innerhalb von notification spezifizieren, z.B. wiki, dann kopiert man den Ordner "userstory" plus Inhalt auf die gleiche Ebene wie "forum" und benennt den Ordner "wiki" und die Datei "Wiki.html".

Elemente

Folgende Elemente stehen für die Gestaltung der Seiten zur Verfügung:

Offene Fragen

Innerhalb eines Abschnitts oder in Form einer Liste oder Tabelle können offene Fragen beschrieben werden. Diese Klasse dient u.a. als Diskussionsgrundlage zwischen SE und RE. Am Ende eines Projekts sollten keine offen Fragen mehr vorhanden sein.

Metriken

Diese Klasse hebt die Metriken hervor, damit sie schnell gefunden werden.

User Story

Mittels Formel: [ALS [BENUTZERROLLE] WILL ICH [DAS ZIEL], so dass [GRUND FÜR DAS ZIEL].] wird die User Story formuliert.

Die Akzeptanztests werden in tabellarischer Form mittels "Vorbedingung", "Aktion" und "Erwartung" definiert.

Formatvorlage

 Eine Formatvorlage kann eine E-Mail-Vorlage sein oder auch ein GUI-Screenshot etc. 
 Die Vorlage kann innerhalb eines pre-Tags stehen, muss aber nicht.
 Die Formatvorlage ist möglichst allumfassend und enthält oft mehr Information
 als im Akzeptanztest definiert.
Mittels der Klasse attention können wichtige Hinweise an die SE gekennzeichnet werden.
