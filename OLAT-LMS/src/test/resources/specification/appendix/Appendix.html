OLAT Dokumentation und Specification by Example mit Concordion

Specification by Example mit Concordion fuer OLAT bedeutet, dass die Beschreibung der Anforderungen mit ihren Akzeptanzkriterien im HTML Format abgelegt und organisiert werden.

Die Requirement Engineers erstellen die Anforderungen als Stories und die zugehoerigen Akzeptanzkriterien werden in Form von konkreten Beispielen definiert.
Die Software Engineers verbinden die Beispiele mit dem Java Akzeptanztestcode, so dass die im HTML definierte Vorbedingungen, Aktionen und Erwarteten Werte im Java Testcode verwendet werden.

Diese Stories-Akzeptanztest Dokumentation ist parallel zum Source.- und Testcode und wird in der Continuous Integration Umgebung (analog zu den Unit Tests) bei jeder Aenderung validiert.

Die Requirements Engineers koennen die Software Engineers bei der Testdatenerstellung unterstuetzen. Dazu fuegen sie neue Werte ein, diese werden dann (bei geschickter Organisation der Stories und Tests) ohne weiteres Zutun ausgefuehrt.

Der aktuelle Austausch betreffend offener Fragen kann gerade bei der Story oder dem Beispiel selbst erfolgen. Fuer den Austausch kann z.B. per Instant Messaging oder Email ein Link zur Frage geschickt werden.
